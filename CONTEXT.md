# CONTEXT.md — Музыкальный Квиз

## Что это
Интерактивный музыкальный квиз на двоих, играется удалённо (каждый со своего компьютера), параллельно с голосовым звонком. Веб-приложение, single HTML file, деплоится на GitHub Pages.

## Техстек
- **Frontend**: Vanilla HTML/CSS/JS, single file (`index.html`)
- **Шрифты**: Unbounded (display) + Onest (body) — Google Fonts
- **Синхронизация**: Firebase Realtime Database (push, не polling)
- **Деплой**: GitHub Pages — `https://sashadupak.github.io/music-quiz/`
- **Репо**: `git@github.com:sashadupak/music-quiz.git`

## Как работает мультиплеер
1. Игрок 1 создаёт комнату → получает 4-буквенный код
2. Игрок 2 открывает ту же ссылку → вводит код → присоединяется
3. Игрок 1 нажимает «Начать» → оба синхронно стартуют
4. Firebase хранит: имена игроков, фазу игры, счёт, ответы с временем
5. Структура в Firebase: `rooms/{ROOM_CODE}/...`
6. Комната автоудаляется через 5 минут после финала

## Структура 7 раундов

| # | Название | Тип | Вес | Механика |
|---|----------|-----|-----|----------|
| 1 | Допой строчку | ⚔️ Соревнование | ×1 | Строчка из песни → 4 варианта продолжения. Скорость ответа даёт бонус (< 5с = +2, < 10с = +1) |
| 2 | Кто это? | ⚔️ Соревнование | ×1 | Описание артиста → 4 варианта. Скорость учитывается |
| 3 | Сопоставь пары | 🤝 Кооператив | ×1 | Оба в одной команде. Сопоставить артистов с фактами. Обсуждают голосом, один кликает |
| 4 | Факт или фейк | ⚔️ Приватный | ×1 | Каждый видит СВОЙ вопрос (разные вопросы!). Не могут подсказать друг другу |
| 5 | Эмодзи-шифр | ⚔️ Соревнование | ×2 | Артист зашифрован эмодзи → 4 варианта. Удвоенные очки |
| 6 | Ставка | 🎰 Риск | ×2 | Факты открываются по одному. Перед ответом ставишь 1-3 очка. Угадал = +ставка×2, нет = -ставка |
| 7 | Финальный блиц | ⚔️ Соревнование | ×3 | 5 быстрых вопросов по 10 сек. УТРОЕННЫЕ очки — может перевернуть всё |

## Дизайн
- Тёмная тема (#0a0a0f)
- Акценты: золотой (#f0c040), красный (#ff6b6b), бирюзовый (#4ecdc4), фиолетовый (#a78bfa)
- Noise overlay на body
- Glow orbs (размытые цветные круги) фоном
- Анимации: fadeUp для интро раундов, shake для неправильных ответов, pulse для индикатора хода, confetti на финале
- Звуки через Web Audio API: клик, правильно (восходящие тоны), неправильно (sawtooth), фанфары, тиканье таймера

## Ключевые принципы контента
- Вопросы НЕ должны быть слишком очевидными — использовать не только топовые треки
- Должен быть интересный процесс, вовлекающий, с подсказками
- Не цель серьёзно мерить конкуренцию, но элементы азарта нужны
- Микс соревновательных и кооперативных раундов
- Последний раунд (×3) должен давать шанс отыграться
- Учитывать скорость ответа в соревновательных раундах
